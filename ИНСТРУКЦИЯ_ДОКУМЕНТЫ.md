# 📄 Инструкция по добавлению документов для автоматической отправки

## 🎯 Как это работает

Система автоматически отправляет документы, когда пользователь задает вопрос по определенной теме. Бот анализирует ключевые слова в вопросе и прикрепляет нужные файлы к ответу.

## 📁 Куда складывать документы

### Структура папок:
```
data/
├── 01_company/     # Общие документы компании
├── 02_hr/          # HR документы (отпуска, больничные)
├── 03_routine/     # Повседневные вопросы (столовая, режим)
├── 04_logistics/   # Логистика и транспорт
├── 05_it/          # IT документы
├── 06_social/      # Социальные вопросы
└── 07_kalendar/    # Календари и графики
```

### Правила именования:
- Используйте латинские буквы и цифры
- Избегайте пробелов - заменяйте их на `_`
- Примеры: `reglament_stolovaya.pdf`, `grafik_vaht.docx`, `pravila_bolnichnogo.docx`

## 🔧 Как добавить новый документ

### Шаг 1: Поместите файл в нужную папку
Скопируйте ваш документ в соответствующую папку внутри `data/`. Например:
- Регламент столовой → `data/03_routine/reglament_stolovaya.pdf`
- График вахт → `data/01_company/grafik_vaht.docx`

### Шаг 2: Откройте файл настроек
Откройте файл: `src/core/document_sender.py`

### Шаг 3: Добавьте правило для документа
Найдите секцию `self.rules = {` и добавьте новое правило:

```python
"название_правила": DocumentRule(
    keywords=["ключевое1", "ключевое2", "ключевое3"],
    document_path="data/папка/имя_файла.pdf",
    description="Описание документа для пользователя",
    file_type="pdf"  # или "docx", "image", "auto"
),
```

#### Пример 1: Регламент столовой

## 📋 Поддерживаемые типы файлов

| Тип файла | Расширения | Описание |
|-----------|-------------|----------|
| Изображения | `.jpg`, `.jpeg`, `.png`, `.gif`, `.bmp`, `.webp` | Отправляются как фото |
| PDF | `.pdf` | Отправляются как документ |
| Документы Word | `.docx`, `.doc` | Отправляются как документ |
| Другие | любые | Отправляются как документ |

## 🎨 Параметры правила

### `keywords` (обязательно)
Список ключевых слов, которые вызывают отправку документа.
- **Совет**: Добавляйте синонимы и разные формы слов
- **Пример**: `["отпуск", "отдых", "каникулы", "выходные", "отпускные"]`

### `document_path` (обязательно)
Путь к файлу относительно корня проекта.
- **Формат**: `data/папка/имя_файла.расширение`
- **Пример**: `data/02_hr/reglament_otpuskov.pdf`

### `description` (обязательно)
Описание, которое увидит пользователь под документом.
- **Пример**: `"Положение об отпусках"`
- **Совет**: Делайте описание понятным и кратким

### `file_type` (опционально)
Тип файла. Если не указано, система определит автоматически.
- **Значения**: `"pdf"`, `"docx"`, `"image"`, `"auto"`
- **По умолчанию**: `"auto"`

## ✅ Проверка работы

Чтобы проверить, что все работает:

1. Задайте вопрос с ключевым словом
2. Бот должен ответить текстом + прикрепить документ
3. Проверьте логи, если что-то не работает

### Примеры вопросов для проверки:
- "Во сколько столовая?" → должен отправить регламент
- "Какое расписание вахт?" → должен отправить график
- "Как оформить отпуск?" → должен отправить положение

## 🛠️ Поиск ошибок

Если документ не отправляется:

1. **Проверьте путь файла**
   ```bash
   # В Windows
   dir "data\03_routine\reglament_stolovaya.pdf"
   
   # В Linux
   ls "data/03_routine/reglament_stolovaya.pdf"
   ```

2. **Проверьте ключевые слова**
   - Убедитесь, что слова из вопроса совпадают с `keywords`
   - Попробуйте разные формы слов

3. **Проверьте логи бота**
   - Ищите сообщения об ошибках
   - Проверьте, что правило загрузилось

4. **Проверьте права доступа**
   - Убедитесь, что бот имеет доступ к файлу
   - Проверьте, что файл не занят другой программой

## 📝 Советы по ключевым словам

### Хорошие примеры:
```python
# Для столовой - много синонимов
keywords=["столовая", "обед", "прием пищи", "еда", "завтрак", "ужин", "ланч"]

# Для вахт - разные формулировки
keywords=["вахта", "график", "расписание", "дежурство", "смены", "рабочее время"]

# Для отпусков - включаем связанные темы
keywords=["отпуск", "отдых", "каникулы", "выходные", "отпускные", "отпускной"]
```

### Плохие примеры:
```python
# Слишком специфично
keywords=["регламент посещения столовой"]

# Слишком мало слов
keywords=["столовая"]

# Опечатки
keywords=["столовая", "обед", "прием пищы"]  # ❌ "пищы" → "пищи"
```
