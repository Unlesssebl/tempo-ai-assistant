# Инструкции по работе с IntraService API

Этот файл является обязательным руководством для любого AI-агента в данном воркспейсе.

## 1. Навигация по документации
- Полная документация разбита на 82 файла: `IntraService_API_v4_47_ЧастьN.md`.
- **ВСЕГДА** сначала обращайся к файлу `IntraService_API_Index.md`, чтобы найти номер нужной части по методу или эндпоинту.
- **ДЛЯ СХЕМ JSON** и обязательных полей используй файл `Schemas.md`.

## 2. Правила авторизации (Auth)
- Используется **Basic Authentication**.
- Заголовок: `Authorization: Basic <base64(login:password)>`.
- *Внимание:* Никогда не сохраняй реальные логины и пароли в коде. Используй переменные окружения.

## 3. Формат запросов и ответов
- **Base URL:** `https://{your-domain}.intraservice.ru/api/`
- **Формат даты:** `YYYY-MM-DD HH:MM`. При парсинге учитывай часовой пояс (подробности в Части 10).
- **ID сущностей:** Почти всегда возвращаются как целые числа (`Int`).
- **Кодировка:** UTF-8.

## 4. Особенности API
- **Поиск и фильтрация:** Поддерживаются параметры `search`, `fields`, `page`, `pagesize`. Детали фильтрации по конкретным полям ищи в Частях 17-19 (для заявок).
- **Файлы:** Загрузка файлов — двухэтапный процесс. Сначала `POST /api/taskfile`, затем передача ID в массив `Files` при создании/изменении заявки.
- **Вложенные данные:** Используй параметр `include` (например, `include=status,priority`), чтобы получить связанные объекты в одном ответе и сэкономить количество запросов.

## 5. Порядок действий при разработке
1. Найти эндпоинт в `IntraService_API_Index.md`.
2. Проверить обязательные поля в `Schemas.md`.
3. При необходимости уточнить детали (фильтры, исключения) в соответствующей "Части" документации.
