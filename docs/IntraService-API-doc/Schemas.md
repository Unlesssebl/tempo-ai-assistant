# Схемы данных IntraService API (Кратко)

Этот файл содержит минимально необходимый набор полей для работы с основными сущностями.

## 1. Создание заявки (POST /api/task)
*См. подробнее в Части 25-26*

| Поле | Тип | Обяз. | Описание |
| :--- | :--- | :---: | :--- |
| `Name` | String | + | Тема заявки |
| `Description` | String | | Описание (тело) заявки |
| `ServiceId` | Int | + | ID сервиса |
| `StatusId` | Int | + | ID начального статуса |
| `PriorityId` | Int | | ID приоритета |
| `TypeId` | Int | | ID типа заявки |
| `CreatorId` | Int | | ID заявителя (если не текущий) |
| `ExecutorIds` | Int[] | | Список ID исполнителей |
| `AssetIds` | Int[] | | Список ID активов |

**Пример JSON:**
```json
{
  "Name": "Проблема с доступом",
  "Description": "Не могу зайти в почту",
  "ServiceId": 10,
  "StatusId": 1
}
```

## 2. Поля ответа (GET /api/task/{id})
*Основные поля, используемые в логике*

- `Id`: Уникальный номер заявки (Int).
- `Name`: Тема.
- `Created`: Дата создания (DateTime).
- `StatusName` / `StatusId`: Информация о статусе.
- `ServiceName` / `ServiceId`: Информация о сервисе.
- `CreatorName`: Имя заявителя.
- `Description`: Полное описание.

## 3. Общие типы данных
- **Даты**: Передаются и принимаются в формате ISO или `YYYY-MM-DD HH:MM`.
- **Файлы**: При создании заявки файлы сначала загружаются через `POST /api/taskfile`, затем полученные ID передаются в массив `Files`.

## 4. Схема Пользователя (User)
- `Id`: ID пользователя.
- `Name`: ФИО.
- `Email`: Электронная почта.
- `CompanyId`: ID компании, к которой привязан пользователь.
- `RoleId`: Системная роль.
